Jenkins Bot - User Guide

1. Introduction

Welcome to the Jenkins Bot User Guide. This guide will help you get started with using the bot to interact with your Jenkins CI/CD pipelines directly from your chat platform. With simple commands and actions, you can manage jobs, check server status, and much more.

2. Getting Started

  • Ensure your Jenkins server is publicly accessible via a URL or IP address.
  • Install the bot on your preferred chat platform (Zoho Cliq, Slack, Microsoft Teams, etc.).
  • Set up the necessary Jenkins plugins (e.g., REST API Plugin and CSRF Protection).
  • Authenticate with your Jenkins server using the bot's login action.

3. Supported Actions

  • login: Save your Jenkins username and password.
  • csrf: Save the CSRF token required for secure actions.
  • url: Save the public Jenkins URL or IP address.
  • Status: Check the Jenkins server status.
  • Job: Retrieve details about a specific Jenkins job.

4. Available Commands

  • /trigger: Trigger a specific Jenkins job.
  • /disable: Disable a specific Jenkins job.
  • /enable: Enable a previously disabled job.
  • /delete: Delete a Jenkins job.

5. Troubleshooting

  • If the bot is unable to connect, ensure the Jenkins URL is publicly accessible.
  • If commands fail, check the provided job name and ensure it matches the job on Jenkins.
  • If authentication issues arise, update your credentials or CSRF token using the login and csrf actions.

6. Security Guidelines

  • Always use HTTPS to secure communication between the bot and Jenkins.
  • Use strong passwords and periodically update the CSRF token.
  • Assign roles and permissions in Jenkins carefully to limit unauthorized actions.
