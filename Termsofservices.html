Service Documentation for Jenkins Bot

1. Overview

The Jenkins Bot is a chat-based tool that simplifies interaction with Jenkins CI/CD pipelines. It allows users to manage jobs, monitor the server, and execute common tasks directly from a chat platform.

2. Features

  • Menu-based actions for quick setup and task management.
  • Command-based job operations like triggering, enabling, disabling, and deleting jobs.
  • Allows checking the Jenkins server status.
  • Supports public Jenkins URLs or IPs only for enhanced accessibility and security.

3. Supported Actions and Commands

Actions (Menu)

  • login: Save the username and password for authentication.
  • csrf: Save the Jenkins CSRF token required for secure actions.
  • url: Save the public URL of your Jenkins server. Only public URLs or IPs are allowed.
  • Status: Retrieve the current status of the Jenkins server.
  • Job: Get details of a specific Jenkins job.

Commands (Chat)

  • /trigger: Trigger a specific Jenkins job.
  • /disable: Disable a specific Jenkins job.
  • /enable: Enable a previously disabled job.
  • /delete: Delete a Jenkins job.

4. Prerequisites

  • Use a public Jenkins URL or IP for bot connectivity. Private/local addresses are not supported.
  • Install necessary Jenkins plugins, such as the REST API Plugin and CSRF Protection enabled.
  • Provide valid credentials, including username, password, and CSRF token.
  • Use the bot with a compatible chat platform like Zoho Cliq, Slack, or Microsoft Teams.

5. Installation

  • Install the bot from the designated repository or package.
  • Configure your Jenkins server with REST API and CSRF plugins.
  • Use the bot's menu to save your Jenkins URL, username, password, and token.
  • Test connectivity and functionality using commands like /trigger or actions like Status.

6. Security Guidelines

  • Use only HTTPS to secure communication between the bot and Jenkins.
  • Set strong passwords and rotate your CSRF tokens periodically.
  • Assign user roles and permissions carefully in Jenkins to limit sensitive operations.

7. Maintenance and Troubleshooting

  • Ensure your Jenkins server has a valid public URL/IP.
  • If the CSRF token validation fails, use the csrf action to update the token.
  • Check the bot's configuration if commands fail to execute. Verify the job name, URL, or credentials.
